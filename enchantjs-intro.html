Chapô

Je cherchais un framework javascript pour faire quelques mini-jeux 2D. La liste étant plutôt longue, j’en ai pris un un peu au hasard. J’avais besoin de quelque chose de léger et super basique. Je me suis tourné vers enchant.js. Je me suis rendu compte qu’il semble malheureusement être laissé à l’abandon depuis à peu près un an (si on s’en réfère au site et à github), mais comme le but n’est pas de développer des jeux très complexes avec webGL etc., mais plus de se (re)faire la main avec javascript, c’est bien suffisant.

Je vais faire ici une petite introduction sur comment utiliser enchant.js en me basant sur le tutoriel officiel. Mais si vous voulez apprendre par des exemples, allez faire un tour sur 9leap.net et code.9leap.net. Le framework étant japonais, les commentaires le sont également… Je fais ça également (ou surtout) pour moi, pour me rappeler comment faire quoi, par qui, etc.

Organisation d’un jeu

Pour faire un jeu avec enchant.js, il faut une page html qui charge la bibliothèque et votre(vos) script(s). Ce dernier va créer une balise div qui contiendra le jeu. (Il est possible de mettre plusieurs jeux sur une même page.) Dans le script, il va y avoir plusieurs types d’objets.

L’objet Game

C’est l’objet qui supervise le jeu, le système qui l’exécute, les fonctionnalités basiques, etc.

Chaque objet Game crée un objet Scene appelé rootScene.

Les objets Scene

Les objets Scene sont une collection d’objets tous affichés en même temps. Les objets Scene sont utilisés pour gérer la visibilité des objets Node, les éléments qu’on affiche pour faire le jeu. Par exemple, quand un jeu a un écran de démarrage, changer de l’écran de démarrage vers l’écran du jeu en lui-même est réalisé en changeant d’objet Scene.

Un jeu peut avoir un seul ou plusieurs objets Scene. Si le jeu est très simple, changer des images ou faire des transitions peut se faire en ajoutant ou supprimant des éléments du rootScene.

Enfin, les objets Scene peuvent être utilisés en couches. Par exemple, on peut avoir un objet Scene qui contient le score et qui sera affiché devant l’objet Scene qui contient le jeu en lui-même.

Les objets Node

Ce sont les éléments visuels du jeu. Par exemple, un objet Sprite qui affiche un dessin ou une image, un objet Label pour afficher du texte, ou encore un objet Map pour afficher une carte.

Le cycle de vie d’un jeu

Quand un jeu enchant.js s’initialise, voilà ce qui se passe :

  1. On initialise enchant.js.
  2. On crée un objet Game
  3. On l’initialise.
    • Différents éléments sont préchargés.
    • Le code du début de jeu est défini.
  4. Le jeu commence.
    • Le jeu attend que le préchargement se finisse.
    • Puis le code du jeu commence à être éxécuté.

Concrètement, ça ressemble à ça : 

1: enchant();
2: 
3: var game = new Game(320, 320);
4: game.preload('preload1.gif', 'preload2.wav');
5: game.onload = function () {
6:   // Game code is written here
7: };
8: 
9: game.start();
